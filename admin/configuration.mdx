## Database Configuration

### Default (SQLite)

The default configuration uses SQLite for development:

```python
# auditoriapp/settings.py:88
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.sqlite3",
        "NAME": BASE_DIR / "db.sqlite3",
    }
}
```

### MySQL/MariaDB (Production)

For production, configure MySQL/MariaDB:

```python
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.mysql",
        "NAME": "auditoriapp_db",
        "USER": "auditoriapp_user",
        "PASSWORD": "secure_password_here",
        "HOST": "localhost",
        "PORT": "3306",
        "OPTIONS": {
            "charset": "utf8mb4",
            "init_command": "SET sql_mode='STRICT_TRANS_TABLES'"
        }
    }
}
```

### Using Environment Variables

Recommended approach for production:

```python
import os
from decouple import config

DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.mysql",
        "NAME": config('DB_NAME'),
        "USER": config('DB_USER'),
        "PASSWORD": config('DB_PASSWORD'),
        "HOST": config('DB_HOST', default='localhost'),
        "PORT": config('DB_PORT', default='3306'),
    }
}
```

Create a `.env` file:

```bash
DB_NAME=auditoriapp_db
DB_USER=auditoriapp_user
DB_PASSWORD=your_secure_password
DB_HOST=localhost
DB_PORT=3306
```

<Note>
  The project includes `python-decouple==3.8` in requirements.txt for environment variable management.
</Note>

## JWT Configuration

### Token Lifetimes

```python
# auditoriapp/settings.py:163
SIMPLE_JWT = {
    'ACCESS_TOKEN_LIFETIME': timedelta(minutes=60),
    'REFRESH_TOKEN_LIFETIME': timedelta(days=7),
    'ROTATE_REFRESH_TOKENS': True,
    'BLACKLIST_AFTER_ROTATION': True,
    'ALGORITHM': 'HS256',
    'SIGNING_KEY': SECRET_KEY,
}
```

| Setting | Default | Description |
|---------|---------|-------------|
| `ACCESS_TOKEN_LIFETIME` | 60 minutes | How long access tokens are valid |
| `REFRESH_TOKEN_LIFETIME` | 7 days | How long refresh tokens are valid |
| `ROTATE_REFRESH_TOKENS` | True | Generate new refresh token on refresh |
| `BLACKLIST_AFTER_ROTATION` | True | Blacklist old refresh tokens |
| `ALGORITHM` | HS256 | JWT signing algorithm |
| `SIGNING_KEY` | SECRET_KEY | Key used to sign tokens |

### Customizing Token Lifetimes

For tighter security, reduce token lifetimes:

```python
SIMPLE_JWT = {
    'ACCESS_TOKEN_LIFETIME': timedelta(minutes=15),  # Shorter access
    'REFRESH_TOKEN_LIFETIME': timedelta(days=1),     # Shorter refresh
    'ROTATE_REFRESH_TOKENS': True,
    'BLACKLIST_AFTER_ROTATION': True,
}
```

For convenience in development:

```python
SIMPLE_JWT = {
    'ACCESS_TOKEN_LIFETIME': timedelta(hours=24),   # Longer for dev
    'REFRESH_TOKEN_LIFETIME': timedelta(days=30),
}
```

## Troubleshooting

### CORS Errors

1. Verify frontend URL is in `CORS_ALLOWED_ORIGINS`
2. Check `CORS_ALLOW_CREDENTIALS` is set correctly
3. Ensure CorsMiddleware is before CommonMiddleware
4. Disable `CORS_ALLOW_ALL_ORIGINS` in production

### JWT Token Issues

1. Verify SECRET_KEY hasn't changed
2. Check token hasn't expired (ACCESS_TOKEN_LIFETIME)
3. Ensure timezone settings are correct
4. Verify SIMPLE_JWT configuration

### Database Connection Errors

1. Verify database credentials in .env
2. Check database service is running
3. Ensure MySQL/PostgreSQL client is installed
4. Test connection with database CLI

### Static Files Not Loading

1. Run `python manage.py collectstatic`
2. Verify STATIC_ROOT and STATIC_URL settings
3. Check web server configuration (nginx/Apache)
4. Ensure WhiteNoise is configured correctly
