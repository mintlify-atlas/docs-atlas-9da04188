title: 'Renditions API'
description: 'API endpoints for managing financial renditions and expense reports'
---

The Renditions API allows you to manage financial renditions (rendiciones) - expense reports that document project expenditures with supporting documentation.

## Base URL

All renditions endpoints are prefixed with:

```
/api/rendiciones/
```

## Authentication

All endpoints require JWT authentication. Include the token in the Authorization header:

```
Authorization: Bearer <your_jwt_token>
```

---

## Renditions

### Rendition States

Renditions follow a review workflow with these possible states:

- `pendiente` - Pending review
- `aprobado` - Approved
- `observado` - With observations/issues
- `rechazado` - Rejected
- `pagado` - Paid

### List Renditions

Retrieve all renditions for projects in the authenticated user's community.

```
GET /api/rendiciones/
```

<Info>
Renditions are automatically filtered based on the authenticated user's community.
</Info>

<ParamField query="proyecto" type="integer">
  Filter by project ID
</ParamField>

<ParamField query="estado" type="string">
  Filter by state (pendiente, aprobado, observado, rechazado, pagado)
</ParamField>

</ResponseField>

<ResponseField name="proyecto" type="integer">
  ID of the associated project
</ResponseField>

</ResponseField>

<ResponseField name="documentos_adjuntos" type="array">

</ResponseField>

<ResponseField name="documentos_ids" type="array">

<ResponseField name="comprobante_traspaso" type="file">

<ResponseField name="observacion" type="text">
  Reviewer observations (read-only)

<ResponseField name="revisor" type="integer">
  ID of the reviewer user (read-only)

<ResponseField name="fecha_revision" type="datetime">

### Create Rendition

Create a new rendition. Documents should be uploaded first using the Documents API.

```
POST /api/rendiciones/
```

<ParamField body="proyecto" type="integer" required>
  ID of the project this rendition belongs to
</ParamField>

<ParamField body="descripcion" type="text">
  Description of the expense
</ParamField>

<ParamField body="monto_rendido" type="decimal" required>
  Amount being submitted
</ParamField>

<ParamField body="numero_documento" type="string">
  Invoice or receipt number
</ParamField>

<ParamField body="fecha_rendicion" type="date" required>
  Date of the rendition (YYYY-MM-DD)
</ParamField>

<ParamField body="documentos_ids" type="array">
  Array of document IDs to attach (documents must be uploaded first)
</ParamField>

<ParamField body="comprobante_traspaso" type="file">
  Payment proof file
</ParamField>

<Info>
Use the "Upload First" pattern: upload documents to `/api/documentos/` first, then include their IDs in `documentos_ids`.
</Info>

<CodeGroup>
```bash cURL
curl -X POST "https://api.auditoriapp.com/api/rendiciones/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "proyecto": 1,
    "descripcion": "Pago a proveedor de materiales",
    "monto_rendido": "4200.00",
    "numero_documento": "FAC-002345",
    "fecha_rendicion": "2024-04-10",
    "documentos_ids": [48, 49]
  }'
```

### Update Rendition

Update an existing rendition.

```
PUT /api/rendiciones/{id}/
PATCH /api/rendiciones/{id}/
```

<ParamField path="id" type="integer" required>
  The unique identifier of the rendition to update
</ParamField>

<Warning>
The `estado`, `observacion`, `revisor`, and `fecha_revision` fields are read-only and can only be modified via the `/revisar/` action.
</Warning>

### Delete Rendition

Delete a rendition.

```
DELETE /api/rendiciones/{id}/
```

## Review Rendition

Review a rendition and change its state. Only available to CPA reviewers and admins.

```
POST /api/rendiciones/{id}/revisar/
```

<ParamField path="id" type="integer" required>
  The unique identifier of the rendition to review
</ParamField>

<ParamField body="estado" type="string" required>
  New state: "aprobado", "observado", or "rechazado"
</ParamField>

<ParamField body="observacion" type="string">
  Review observations or comments
</ParamField>

<Warning>
Only users with role "cpa", "admin", or "CPA Revisor" can review renditions.
</Warning>

<CodeGroup>
```bash cURL - Approve
curl -X POST "https://api.auditoriapp.com/api/rendiciones/1/revisar/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "estado": "aprobado",
    "observacion": "Documentación completa y correcta"
  }'
```

```bash cURL - Reject with Observations
curl -X POST "https://api.auditoriapp.com/api/rendiciones/2/revisar/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "estado": "observado",
    "observacion": "Falta comprobante de pago, favor adjuntar"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.auditoriapp.com/api/rendiciones/1/revisar/', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    estado: 'aprobado',
    observacion: 'Documentación completa y correcta'
  })
});
```


## Workflow

Typical rendition workflow:

1. **Create**: Community admin creates rendition with supporting documents
2. **Submit**: Rendition starts in `pendiente` state
3. **Review**: CPA/Auditor reviews using `/revisar/` endpoint
   - Approve → `aprobado`
   - Request changes → `observado`
   - Reject → `rechazado`
4. **Payment**: Once approved, treasurer uploads payment proof
5. **Mark Paid**: State changes to `pagado` after payment confirmation

## Permissions

- All endpoints require authentication
- Users can only access renditions from projects in their community
- Only CPA, CPA Revisor, and admin users can review renditions
- Filtering is available by project and state
