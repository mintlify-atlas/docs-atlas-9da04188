The Projects API allows you to manage projects (proyectos) and their progress reports throughout their lifecycle, from draft to closure.

## Base URL

All projects endpoints are prefixed with:

```
/api/
```

## Authentication

All endpoints require JWT authentication. Include the token in the Authorization header:

```
Authorization: Bearer <your_jwt_token>
```

---

## Projects

### Project States

Projects follow a workflow with these possible states:

- `borrador` - Draft
- `revision_interna` - Internal Review
- `revision_cpa` - CPA Review
- `observaciones` - With Observations
- `aprobado` - Approved
- `ejecucion` - In Execution
- `cierre` - Closure

### Create Project

Create a new project. Uses "Upload First" pattern - documents must be uploaded separately before creating the project.

```
POST /api/proyectos/
```

<ParamField body="comunidad" type="integer" required>
  ID of the community (auto-assigned for Admin Comunidad users)
</ParamField>

<ParamField body="periodo" type="integer" required>
  ID of the associated period
</ParamField>

<ParamField body="nombre" type="string" required>
  Name of the project
</ParamField>

<ParamField body="descripcion" type="text">
  Detailed description
</ParamField>

<ParamField body="fecha_inicio" type="date">
  Start date (YYYY-MM-DD)
</ParamField>

<ParamField body="fecha_fin" type="date">
  End date (YYYY-MM-DD)
</ParamField>

<ParamField body="presupuesto_total" type="decimal" default="0.00">
  Total budget
</ParamField>

<ParamField body="objetivos" type="text">
  Main objectives
</ParamField>

<ParamField body="justificacion" type="text">
  Project justification
</ParamField>

<ParamField body="beneficiarios_estimados" type="integer" default="0">
  Estimated number of beneficiaries
</ParamField>

<ParamField body="quorum_asamblea" type="integer" default="0">
  Assembly attendees (must be greater than 0)
</ParamField>

<ParamField body="firma_presidente" type="boolean" default={false}>
  President's digital approval
</ParamField>

<ParamField body="documentos_ids" type="array">
  Array of document IDs previously uploaded (for Acta, Cotizaciones, etc.)

<Info>
Documents should be uploaded to `/api/documentos/` first, then their IDs included in `documentos_ids`.

### Update Project

Update an existing project.

```
PUT /api/proyectos/{id}/
PATCH /api/proyectos/{id}/
```

<ParamField path="id" type="integer" required>
  The unique identifier of the project to update
</ParamField>

<Warning>
The `estado` and `fecha_firma_presidente` fields are read-only and cannot be updated directly. Use the `/cambiar_estado/` action to change project state.
</Warning>

### Delete Project

Delete a project.

```
DELETE /api/proyectos/{id}/
```

## Change Project State

Change the state of a project following the workflow rules.

```
POST /api/proyectos/{id}/cambiar_estado/
```

<ParamField path="id" type="integer" required>
  The unique identifier of the project
</ParamField>

<ParamField body="nuevo_estado" type="string" required>
  The new state to transition to
</ParamField>

<ParamField body="comentario" type="string">
  Optional comment about the state change
</ParamField>

### State Transition Rules

<Warning>
State transitions are restricted based on user roles and current state:
</Warning>

1. **Borrador → Revisión Interna**: Any authorized user
2. **Observaciones → Revisión Interna**: Any authorized user
3. **Revisión Interna → Revisión CPA**: Only President or Directorio
4. **Revisión CPA → Aprobado**: Only CPA
5. **Revisión CPA → Observaciones**: Only CPA

<CodeGroup>
```bash cURL
curl -X POST "https://api.auditoriapp.com/api/proyectos/1/cambiar_estado/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "nuevo_estado": "revision_cpa",
    "comentario": "Proyecto listo para revisión CPA"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.auditoriapp.com/api/proyectos/1/cambiar_estado/', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    nuevo_estado: 'revision_cpa',
    comentario: 'Proyecto listo para revisión CPA'
  })
});
```
</CodeGroup>

<ResponseExample>
```json Success
{
  "message": "Estado cambiado a revision_cpa",
  "estado_anterior": "revision_interna",
  "estado_nuevo": "revision_cpa"
}
```
</ResponseExample>

<ResponseExample>
```json Error - Insufficient Permissions
{
  "error": "Solo Presidente o Directorio pueden enviar a CPA"
}
```
</ResponseExample>

### Create Progress Report

Create a new progress report for a project.

```
POST /api/reportes-avance/
```

<ParamField body="proyecto" type="integer" required>
  ID of the project
</ParamField>

<ParamField body="porcentaje_avance" type="integer" required>
  Physical progress percentage (0-100)
</ParamField>

<ParamField body="observaciones" type="text" required>
  Progress observations
</ParamField>

<ParamField body="foto_avance" type="file">
  Progress photo (image file)
</ParamField>

<Warning>
Only users with role "ITO" (Inspector Técnico de Obras) or "admin" can create progress reports.
</Warning>

<CodeGroup>
```bash cURL
curl -X POST "https://api.auditoriapp.com/api/reportes-avance/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "proyecto=1" \
  -F "porcentaje_avance=75" \
  -F "observaciones=Avance en muros perimetrales" \
  -F "foto_avance=@/path/to/photo.jpg"
```

## Permissions

- All endpoints require authentication
- Users can only access projects from their associated community
- State changes have specific role requirements (CPA, President, Directorio)
- Progress reports can only be created by ITO or admin users
