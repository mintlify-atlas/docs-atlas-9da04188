## Overview

Renditions (`Rendicion`) are expense accountability submissions tied to projects. They track actual expenditures with supporting documentation through a rigorous review process from submission to payment.

## Rendicion Model

The rendition model captures expense details and review workflow:

```python
class Rendicion(models.Model):
    ESTADO_RENDICION = [
        ('pendiente', 'Pendiente'),
        ('aprobado', 'Aprobado'),
        ('observado', 'Con Observaciones'),
        ('rechazado', 'Rechazado'),
        ('pagado', 'Pagado'),
    ]
    
    proyecto = models.ForeignKey('proyectos.Proyecto', on_delete=models.CASCADE)
    descripcion = models.TextField(blank=True, null=True)
    monto_rendido = models.DecimalField(max_digits=12, decimal_places=2)
    numero_documento = models.CharField(max_length=50, help_text="Número de factura o boleta")
    fecha_rendicion = models.DateField()
    documentos_adjuntos = models.ManyToManyField('documentos.Documento', 
                                                related_name='rendiciones_adjuntas')
    
    # Review fields
    estado = models.CharField(max_length=20, choices=ESTADO_RENDICION, default='pendiente')
    observacion = models.TextField(blank=True, null=True)
    revisor = models.ForeignKey('usuarios.CustomUser', on_delete=models.SET_NULL, 
                               null=True, related_name='rendiciones_revisadas')
    fecha_revision = models.DateTimeField(null=True, blank=True)
    comprobante_traspaso = models.FileField(upload_to='comprobantes_pago/', null=True, blank=True)
```

## Rendition States

Renditions progress through five states during their lifecycle:

<Steps>
  <Step title="Pendiente (Pending)">
    Initial state upon submission. Awaiting CPA review.
    
    ```python
    rendicion = Rendicion.objects.create(
        proyecto=proyecto,
        monto_rendido=1500000,
        estado='pendiente'
    )
    ```
  </Step>
  
  <Step title="Aprobado (Approved)">
    CPA has validated the expense. Ready for payment processing.
    
    <Note>Approved renditions count toward `total_rendido_aprobado` in dashboard KPIs.</Note>
  </Step>
  
  <Step title="Observado (With Observations)">
    CPA requires corrections or additional documentation. Submitter must address observations and resubmit.
    
    <Warning>Observed renditions generate alerts on the dashboard until resolved.</Warning>
  </Step>
  
  <Step title="Rechazado (Rejected)">
    CPA has rejected the rendition. No payment will be processed.
  </Step>
  
  <Step title="Pagado (Paid)">
    Payment has been transferred to the provider. Final state.
    
    <Note>Only 'pagado' renditions are counted in budget consumption calculations.</Note>
  </Step>
</Steps>

## Review Workflow

CPA or authorized auditors can review renditions:

```python
@action(detail=True, methods=['post'])
def revisar(self, request, pk=None):
    rendicion = self.get_object()
    user = request.user
    
    # Validate Role (Only CPA or Admin)
    if user.rol not in ['cpa', 'admin', 'CPA Revisor']:
        return Response({'error': 'No tiene permisos para revisar'}, status=403)
    
    nuevo_estado = request.data.get('estado')
    observacion = request.data.get('observacion', '')
    
    if nuevo_estado not in ['aprobado', 'observado', 'rechazado']:
        return Response({'error': 'Estado inválido'}, status=400)
    
    rendicion.estado = nuevo_estado
    rendicion.observacion = observacion
    rendicion.revisor = user
    rendicion.fecha_revision = timezone.now()
    rendicion.save()
    
    return Response({'status': 'ok', 'estado': rendicion.estado})
```

<Warning>
  **Permission Enforcement**: Only users with `rol` in `['cpa', 'admin', 'CPA Revisor']` can execute the `revisar` action.
</Warning>

## Permissions & Scoping

Renditions are scoped to the user's community through the project relationship:

```python
class RendicionViewSet(viewsets.ModelViewSet):
    permission_classes = [IsAuthenticated]
    
    def get_queryset(self):
        user = self.request.user
        if hasattr(user, 'comunidad') and user.comunidad:
            return Rendicion.objects.filter(proyecto__comunidad=user.comunidad)
        return Rendicion.objects.none()
```

<Note>
  The queryset uses a double-underscore lookup `proyecto__comunidad` to filter renditions by the associated project's community.
</Note>

## Validation Rules

<CardGroup cols={2}>
  <Card title="Budget Limits" icon="exclamation-triangle">
    Ensure `monto_rendido` doesn't cause project to exceed its `presupuesto_total`. Validate before creation:
    
    ```python
    total_rendido = proyecto.rendicion_set.aggregate(
        Sum('monto_rendido')
    )['monto_rendido__sum'] or 0
    
    if total_rendido + new_monto > proyecto.presupuesto_total:
        raise ValidationError("Excede presupuesto del proyecto")
    ```
  </Card>
  
  <Card title="Document Evidence" icon="file-invoice">
    Frontend should require at least one document attachment. Backend can enforce:
    
    ```python
    if not documentos_ids:
        raise ValidationError("Debe adjuntar al menos un documento")
    ```
  </Card>
  
  <Card title="Date Validation" icon="calendar-xmark">
    `fecha_rendicion` should be within the project's period dates:
    
    ```python
    if fecha_rendicion < proyecto.periodo.fecha_inicio or \
       fecha_rendicion > proyecto.periodo.fecha_fin:
        raise ValidationError("Fecha fuera del periodo del proyecto")
    ```
  </Card>
  
  <Card title="State Transitions" icon="code-branch">
    Only allow state changes through the `revisar` action. Direct updates to `estado` should be read-only:
    
    ```python
    read_only_fields = ['estado', 'observacion', 'revisor', 'fecha_revision']
    ```
  </Card>
</CardGroup>

## Payment Processing

When a rendition is paid, upload the payment proof:

```python
# Update to 'pagado' state (manual or via custom action)
rendicion.estado = 'pagado'
rendicion.comprobante_traspaso = uploaded_file
rendicion.save()

# Update project's total_rendido
proyecto.total_rendido = proyecto.rendicion_set.filter(
    estado='pagado'
).aggregate(Sum('monto_rendido'))['monto_rendido__sum'] or 0
proyecto.save()
```

<Note>
  The `comprobante_traspaso` field stores the bank transfer receipt or payment confirmation document.
</Note>

## Best Practices

1. **Upload Documents First**: Use the "Upload First" pattern. Upload documents to `/api/documentos/`, then use the returned IDs in `documentos_ids`.

2. **Clear Descriptions**: Provide detailed `descripcion` explaining the expense purpose and relation to the project.

3. **Accurate Document Numbers**: Always include `numero_documento` (invoice/receipt number) for audit trails.

4. **Timely Reviews**: CPAs should review pending renditions promptly to avoid blocking project execution.

5. **Observation Follow-up**: When renditions are marked 'observado', address the `observacion` comments and resubmit quickly.

6. **Payment Proof**: Always upload `comprobante_traspaso` when marking as 'pagado' for complete accountability.
