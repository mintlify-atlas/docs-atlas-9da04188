## Overview

The document management system provides centralized file storage with associations to projects and renditions. It implements an "Upload First" pattern where documents are uploaded independently, then linked to entities.

## Documento Model

Simple and flexible document structure:

```python
class Documento(models.Model):
    fecha_subida = models.DateTimeField(auto_now_add=True)
    proyecto = models.ForeignKey('proyectos.Proyecto', on_delete=models.SET_NULL, 
                                null=True, blank=True)
    nombre = models.CharField(max_length=255)
    tipo = models.CharField(max_length=50)
    archivo = models.FileField(upload_to='documentos/')
    descripcion = models.TextField(blank=True)
```

## Upload First Pattern

Documents are uploaded before entity creation, then linked by ID:

<Steps>
  <Step title="Upload Document">
    Submit file with metadata to the documents endpoint.
    
    ```bash
    POST /api/documentos/
    Content-Type: multipart/form-data
    
    {
      "nombre": "Acta Asamblea Marzo 2024",
      "tipo": "Acta",
      "descripcion": "Aprobación proyecto plaza",
      "archivo": <file>
    }
    ```
    
    Response:
    ```json
    {
      "id": 12,
      "nombre": "Acta Asamblea Marzo 2024",
      "tipo": "Acta",
      "archivo": "/media/documentos/acta_marzo_2024.pdf",
      "fecha_subida": "2024-03-15T10:30:00Z",
      "proyecto": null
    }
    ```
  </Step>
  
  <Step title="Create Entity with Document IDs">
    Include the document IDs when creating projects or renditions.
    
    ```bash
    POST /api/proyectos/proyectos/
    {
      "nombre": "Mejoramiento Plaza",
      "documentos_ids": [12, 13, 14],
      ...
    }
    ```
  </Step>
  
  <Step title="Documents Get Linked">
    The serializer links documents to the project.
    
    ```python
    def create(self, validated_data):
        documentos_ids = validated_data.pop('documentos_ids', [])
        proyecto = Proyecto.objects.create(**validated_data)
        
        if documentos_ids:
            docs = Documento.objects.filter(id__in=documentos_ids)
            for doc in docs:
                doc.proyecto = proyecto
                doc.save()
        
        return proyecto
    ```
  </Step>
</Steps>

<Note>
  This pattern prevents orphaned documents and ensures files exist before entity creation fails validation.
</Note>

## File Types & Validation

The `FileField` accepts any file type by default. Add validation in the serializer:

```python
class DocumentoSerializer(serializers.ModelSerializer):
    class Meta:
        model = Documento
        fields = '__all__'
    
    def validate_archivo(self, value):
        # Validate file size (e.g., max 10MB)
        if value.size > 10 * 1024 * 1024:
            raise serializers.ValidationError("El archivo no puede exceder 10MB")
        
        # Validate file extension
        allowed_extensions = ['.pdf', '.jpg', '.jpeg', '.png', '.doc', '.docx', '.xls', '.xlsx']
        ext = os.path.splitext(value.name)[1].lower()
        if ext not in allowed_extensions:
            raise serializers.ValidationError(f"Tipo de archivo no permitido: {ext}")
        
        return value
```

    - **JPEG/PNG**: For photos and visual evidence

    - **Documents**: Up to 10MB per file

## Associations with Projects

Projects access their documents via reverse ForeignKey:

```python
# Get all documents for a project
documentos = proyecto.documento_set.all()

# Get specific document types
acta = proyecto.documento_set.filter(tipo='Acta').first()
cotizaciones = proyecto.documento_set.filter(tipo='Cotizaciones')
```

Serializer methods provide convenient URL access:

```python
class ProyectoSerializer(serializers.ModelSerializer):
    acta_url = serializers.SerializerMethodField()
    cotizaciones_urls = serializers.SerializerMethodField()
    elegido_url = serializers.SerializerMethodField()
    
    def get_acta_url(self, obj):
        request = self.context.get('request')
        doc = obj.documento_set.filter(tipo='Acta').last()
        if doc and doc.archivo:
            return request.build_absolute_uri(doc.archivo.url)
        return None
    
    def get_cotizaciones_urls(self, obj):
        request = self.context.get('request')
        docs = obj.documento_set.filter(tipo='Cotizaciones')
        return [request.build_absolute_uri(doc.archivo.url) 
                for doc in docs if doc.archivo]
```

## Associations with Renditions

Renditions use ManyToMany for multiple document attachments:

```python
class Rendicion(models.Model):
    documentos_adjuntos = models.ManyToManyField(
        'documentos.Documento', 
        related_name='rendiciones_adjuntas'
    )
```

Access renditions from a document:

```python
# Get all renditions using this document
rendiciones = documento.rendiciones_adjuntas.all()
```

Serializer handles the relationship:

```python
class RendicionSerializer(serializers.ModelSerializer):
    documentos_adjuntos = DocumentoSerializer(many=True, read_only=True)
    
    documentos_ids = serializers.PrimaryKeyRelatedField(
        many=True,
        write_only=True,
        queryset=Documento.objects.all(),
        source='documentos_adjuntos'
    )
```

## Frontend Integration

### Upload Component Example

```jsx
import { apiFetch } from '../utils/api';

async function uploadDocument(file, tipo, nombre, descripcion) {
  const formData = new FormData();
  formData.append('archivo', file);
  formData.append('tipo', tipo);
  formData.append('nombre', nombre);
  formData.append('descripcion', descripcion);
  
  const response = await apiFetch('/documentos/', {
    method: 'POST',
    body: formData,
    // Don't set Content-Type - browser sets it with boundary
  });
  
  if (response.ok) {
    const documento = await response.json();
    return documento.id; // Use this ID in documentos_ids
  }
  
  throw new Error('Upload failed');
}
```

## Storage Configuration

Documents are stored using Django's `FileField`:

```python
# In settings.py
MEDIA_URL = '/media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')
```

Files are saved to `media/documentos/` by default:

```python
archivo = models.FileField(upload_to='documentos/')
```

<Note>
  For production, configure cloud storage (S3, Google Cloud Storage) using `django-storages` to handle large file volumes and backups.
</Note>

## Best Practices

<CardGroup cols={2}>
  <Card title="Descriptive Names" icon="pen-to-square">
    Use clear, descriptive names like "Factura-Materiales-Marzo-2024.pdf" instead of generic "documento.pdf".
  </Card>
  
  <Card title="Consistent Tipos" icon="layer-group">
    Define a standard set of document types and use them consistently. Consider using choices in the model:
    ```python
    TIPO_CHOICES = [
        ('Acta', 'Acta'),
        ('Factura', 'Factura'),
        ('Boleta', 'Boleta'),
        ...
    ]
    tipo = models.CharField(max_length=50, choices=TIPO_CHOICES)
    ```
  </Card>
  
  <Card title="Pre-Upload Validation" icon="shield-check">
    Validate file size and type on the frontend before uploading to provide better UX and reduce server load.
  </Card>
  
  <Card title="Cleanup Orphans" icon="trash">
    Periodically clean up documents with `proyecto=null` and no rendition associations that are older than a threshold (e.g., 30 days).
  </Card>
</CardGroup>

## Security Considerations

1. **File Type Validation**: Implement strict validation to prevent malicious file uploads.

2. **Access Control**: Add permission checks to ensure users can only access documents from their community.

3. **Virus Scanning**: Consider integrating antivirus scanning for uploaded files in production.

4. **Secure URLs**: Use signed URLs or authentication-required endpoints for sensitive documents.

5. **Audit Trail**: Log document access and modifications for compliance.
