Auditoriapp implements a role-based access control (RBAC) system with multi-tenancy support. Each user is assigned a single role that determines their capabilities within the system.

The platform supports seven distinct user roles, defined in the `CustomUser` model:

## Role Definitions

Roles are stored in the `usuarios.models.CustomUser` model:

```python
class CustomUser(AbstractUser):
    rol = models.CharField(max_length=20, choices=[
        ('admin', 'Admin'),
        ('auditor', 'Auditor'),
        ('usuario', 'Usuario'),
        ('directorio', 'Directorio Comunidad'),
        ('presidente', 'Presidente Comunidad'),
        ('ito', 'ITO / Responsable'),
        ('cpa', 'CPA Revisor')
    ], default='usuario')
    comunidad = models.ForeignKey('comunidades.Comunidad', ...)
    es_auditor = models.BooleanField(default=False)
```

## Multi-Tenancy Model

### Community Scoping

Most users are assigned to a single community via the `comunidad` foreign key. This enforces data isolation:

```python
def get_queryset(self):
    user = self.request.user
    if hasattr(user, 'comunidad') and user.comunidad:
        return Proyecto.objects.filter(comunidad=user.comunidad)
    return Proyecto.objects.none()
```

**Source:** `backend/proyectos/views.py:18-22`

### Cross-Community Access

Only **Auditor** roles can access data across all communities for oversight purposes. Regular users see only their community's data.

## Role Hierarchy

### Approval Authority

Project approval follows a strict hierarchy:

1. **ITO** - Reports physical progress
2. **Admin** - Creates and manages projects
3. **Presidente/Directorio** - Sends projects to CPA review
4. **CPA** - Final approval or rejection

### State Transition Rules

From `backend/proyectos/views.py:41-63`:

- **CPA approval** (line 42-46): Only `cpa` role can approve projects
- **Send to CPA** (line 54-58): Only `presidente`, `directorio`, or `admin` can advance to CPA review
- **Issue observations** (line 61-63): Only `cpa` can return projects with observations

## Role Assignment

### During User Creation

Admins can create users with specific roles:

```python
@api_view(['POST'])
@permission_classes([IsAuthenticated])
def create_user_view(request):
    if request.user.rol not in ['Admin Consejo', 'Admin Comunidad']:
        return Response({'error': 'No tienes permisos para crear usuarios'}, 
                       status=status.HTTP_403_FORBIDDEN)
```

**Source:** `backend/usuarios/views.py:69-75`

### Community Assignment

Community admins automatically assign new users to their own community:

```python
if request.user.rol == 'Admin Comunidad':
    serializer.validated_data['comunidad'] = request.user.comunidad
```
