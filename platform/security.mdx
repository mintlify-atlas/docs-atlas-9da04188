## Overview

Auditoriapp implements **JWT (JSON Web Token) authentication** with **role-based access control (RBAC)** to secure API endpoints and enforce granular permissions across different user types.

```python
    'REFRESH_TOKEN_LIFETIME': timedelta(days=7),
    'ROTATE_REFRESH_TOKENS': True,
    'BLACKLIST_AFTER_ROTATION': True,
    'ALGORITHM': 'HS256', 
    'SIGNING_KEY': SECRET_KEY,
}
```

<Info>
  - **Access tokens** expire after 60 minutes
  - **Refresh tokens** are valid for 7 days
  - Tokens are rotated on refresh for enhanced security
  - Old tokens are blacklisted after rotation
</Info>

### User Roles Explained

<AccordionGroup>
  <Accordion title="admin - System Administrator">
    Full system access, can manage all communities and users. Typically used for platform administrators.
  </Accordion>
  
  <Accordion title="auditor - External Auditor">
    Can review projects and rendiciones across communities. Special flag `es_auditor` provides additional permissions.
  </Accordion>
  
  <Accordion title="usuario - General User">
    Basic community member with read access to projects and limited write permissions.
  </Accordion>
  
  <Accordion title="directorio - Board Member">
    Community board member who can approve projects for CPA review and participate in governance.
  </Accordion>
  
  <Accordion title="presidente - Community President">
    Legal representative who can digitally sign project approvals and send projects to CPA review.
  </Accordion>
  
  <Accordion title="ito - Technical Officer">
    Responsible for reporting physical project progress and uploading technical documentation.
  </Accordion>
  
  <Accordion title="cpa - Certified Public Accountant">
    External reviewer who can approve projects, review rendiciones, and issue observations.
  </Accordion>
</AccordionGroup>

## Authentication Endpoints

### Token Obtain (Login)

```python
# usuarios/views.py:29-61
@api_view(['POST'])
@permission_classes([AllowAny])
def login_view(request):
    username = request.data.get('username')
    password = request.data.get('password')
    
    if not username or not password:
        return Response(
            {'error': 'Username y password son requeridos'}, 
            status=status.HTTP_400_BAD_REQUEST
        )
    
    user = authenticate(username=username, password=password)
    if user:
        serializer = CustomTokenObtainPairSerializer()
        tokens = serializer.get_token(user)
        
        return Response({
            'access': str(tokens.access_token),
            'refresh': str(tokens),
            'user': {
                'id': str(user.id),
                'username': user.username,
                'nombre': user.nombre,
                'email': user.email,
                'rol': user.rol,
                'comunidad': {
                    'id': str(user.comunidad.id) if user.comunidad else None,
                    'nombre': user.comunidad.nombre if user.comunidad else None
                } if user.comunidad else None
            }
        })
    else:
        return Response(
            {'error': 'Credenciales inválidas'}, 
            status=status.HTTP_401_UNAUTHORIZED
        )
```

## Role-Based Permissions

### Custom Permission Classes

```python
# usuarios/permissions.py:1-26
from rest_framework import permissions

class IsAdmin(permissions.BasePermission):
    def has_permission(self, request, view):
        return request.user and request.user.is_authenticated and request.user.rol == 'admin'

class IsAuditor(permissions.BasePermission):
    def has_permission(self, request, view):
        return request.user and request.user.is_authenticated and (
            request.user.rol == 'auditor' or request.user.es_auditor
        )

class IsDirectorio(permissions.BasePermission):
    def has_permission(self, request, view):
        return request.user and request.user.is_authenticated and request.user.rol == 'directorio'

class IsPresidente(permissions.BasePermission):
    def has_permission(self, request, view):
        return request.user and request.user.is_authenticated and request.user.rol == 'presidente'

class IsITO(permissions.BasePermission):
    def has_permission(self, request, view):
        return request.user and request.user.is_authenticated and request.user.rol == 'ito'

class IsCPA(permissions.BasePermission):
    def has_permission(self, request, view):
        return request.user and request.user.is_authenticated and request.user.rol == 'cpa'
```

## Workflow Permissions

### Project State Transitions

Different roles can perform specific state transitions:

```python
# proyectos/views.py:30-83
@action(detail=True, methods=['post'])
def cambiar_estado(self, request, pk=None):
    proyecto = self.get_object()
    nuevo_estado = request.data.get('nuevo_estado')
    comentario = request.data.get('comentario', '')

    # 1. Only CPA can approve projects
    if nuevo_estado == 'aprobado':
        if request.user.rol != 'cpa':
            return Response(
                {'error': 'Solo el CPA puede aprobar proyectos'}, 
                status=status.HTTP_403_FORBIDDEN
            )
        if proyecto.estado != 'revision_cpa':
            return Response(
                {'error': 'El proyecto debe estar en revisión por el CPA'}, 
                status=status.HTTP_400_BAD_REQUEST
            )

    # 2. Draft -> Internal Review
    elif nuevo_estado == 'revision_interna':
        if proyecto.estado not in ['borrador', 'observaciones']:
            return Response(
                {'error': 'Transición no válida'}, 
                status=status.HTTP_400_BAD_REQUEST
            )

    # 3. Internal Review -> CPA Review (President or Board only)
    elif nuevo_estado == 'revision_cpa':
        if request.user.rol not in ['presidente', 'directorio', 'admin']:
            return Response(
                {'error': 'Solo Presidente o Directorio pueden enviar a CPA'}, 
                status=status.HTTP_403_FORBIDDEN
            )
        if proyecto.estado != 'revision_interna':
            return Response(
                {'error': 'El proyecto debe haber pasado la revisión interna'}, 
                status=status.HTTP_400_BAD_REQUEST
            )

    # 4. CPA -> Observations
    elif nuevo_estado == 'observaciones':
        if request.user.rol != 'cpa':
            return Response(
                {'error': 'Solo el CPA puede emitir observaciones'}, 
                status=status.HTTP_403_FORBIDDEN
            )

    # Update state and create history
    estado_anterior = proyecto.estado
    proyecto.estado = nuevo_estado
    proyecto.save()

    HistorialEstado.objects.create(
        proyecto=proyecto,
        estado_anterior=estado_anterior,
        estado_nuevo=nuevo_estado,
        usuario=request.user,
        comentario=comentario
    )

    return Response({
        'message': f'Estado cambiado a {nuevo_estado}',
        'estado_anterior': estado_anterior,
        'estado_nuevo': nuevo_estado
    })
```

## User Creation Permissions

```python
# usuarios/views.py:69-86
@api_view(['POST'])
@permission_classes([IsAuthenticated])
def create_user_view(request):
    # Only Admin Consejo or Admin Comunidad can create users
    if request.user.rol not in ['Admin Consejo', 'Admin Comunidad']:
        return Response(
            {'error': 'No tienes permisos para crear usuarios'}, 
            status=status.HTTP_403_FORBIDDEN
        )
    
    serializer = CreateUserSerializer(data=request.data)
    if serializer.is_valid():
        # Community admin can only create users in their community
        if request.user.rol == 'Admin Comunidad':
            serializer.validated_data['comunidad'] = request.user.comunidad
        
        user = serializer.save()
        return Response(UserSerializer(user).data, status=status.HTTP_201_CREATED)
    
    return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)
```

## Common Security Patterns

### Check Authentication

```python
@api_view(['GET'])
@permission_classes([IsAuthenticated])
def protected_view(request):
    # request.user is guaranteed to be authenticated
    pass
```

### Check Specific Role

```python
if request.user.rol != 'cpa':
    return Response(
        {'error': 'Permission denied'}, 
        status=status.HTTP_403_FORBIDDEN
    )
```

### Check Community Assignment

```python
if not hasattr(user, 'comunidad') or not user.comunidad:
    return Response(
        {"error": "Usuario sin comunidad asignada"}, 
        status=400
    )
```

### Validate Role for Action

```python
if request.user.rol not in ['presidente', 'directorio', 'admin']:
    return Response(
        {'error': 'Insufficient permissions'}, 
        status=status.HTTP_403_FORBIDDEN
    )
```
