## What is Auditoriapp?

Auditoriapp is a **multi-tenant SaaS platform** designed for auditing, traceability, and transparent fund management for community projects across multiple beneficiary organizations. The system replaces manual and opaque workflows (Excel spreadsheets and PDFs) with a centralized digital solution that ensures **data isolation** between communities while providing real-time oversight tools for auditors.

## The Problem It Solves

Traditional community fund management faces critical challenges:

- **Manual processes**: Excel spreadsheets and PDF documents create inefficiency and errors
- **Lack of transparency**: Difficult to track fund allocation and usage across projects
- **No audit trail**: Missing proper documentation and accountability mechanisms
- **Data silos**: Each community manages data separately with no centralized oversight
- **Document management chaos**: Physical receipts, invoices, and evidence scattered across files

Auditoriapp transforms this workflow into a modern, digital, and transparent system.

## Key Features

### Multi-Tenant Architecture

The core of the system is its **logical multi-tenant architecture**, ensuring each organization accesses only their own data:

```python title="backend/comunidades/models.py"
class Comunidad(models.Model):
    nombre = models.CharField(max_length=255)
    activa = models.BooleanField(default=True)
    creada_en = models.DateTimeField(auto_now_add=True)
    usuario = models.ForeignKey('usuarios.CustomUser', on_delete=models.CASCADE, 
                                null=True, blank=True, related_name='comunidades')

    def __str__(self):
        return self.nombre
```

Data isolation is implemented at the ORM level through filters that segregate information for each community entity.

### Role-Based Access Control (RBAC)

The platform implements comprehensive role-based security:

```python title="backend/usuarios/models.py"
class CustomUser(AbstractUser):
    nombre = models.CharField(max_length=100, blank=True)
    comunidad = models.ForeignKey('comunidades.Comunidad', on_delete=models.CASCADE, 
                                  null=True, blank=True)
    rol = models.CharField(max_length=20, choices=[
        ('admin', 'Admin'),
        ('auditor', 'Auditor'),
        ('usuario', 'Usuario'),
        ('directorio', 'Directorio Comunidad'),
        ('presidente', 'Presidente Comunidad'),
        ('ito', 'ITO / Responsable'),
        ('cpa', 'CPA Revisor')
    ], default='usuario')
    telefono = models.CharField(max_length=20, blank=True)
    es_auditor = models.BooleanField(default=False)
```

**Key Roles:**

- **Admin Comunidad**: Complete management of community projects and financial accountability reports
- **Auditor Central**: Cross-community visibility for validation and control
- **CPA Revisor**: Review and approve financial renditions
- **ITO/Responsable**: Technical oversight and project monitoring
- **Presidente Comunidad**: Legal representative with approval authority

## Project Workflow States

```python title="backend/proyectos/models.py"
ESTADO_CHOICES = [
    ('borrador', 'Borrador'),
    ('revision_interna', 'En Revisión Interna'),
    ('revision_cpa', 'En Revisión CPA'),
    ('observaciones', 'Con Observaciones'),
    ('aprobado', 'Aprobado'),
    ('ejecucion', 'En Ejecución'),
    ('cierre', 'Cierre'),
]
```

Projects flow through these states with complete audit trails:

1. **Borrador**: Initial draft stage
2. **Revisión Interna**: Internal community review
3. **Revisión CPA**: External auditor review
4. **Observaciones**: Requires corrections
5. **Aprobado**: Approved for execution
6. **Ejecución**: Active project implementation
7. **Cierre**: Project completed and closed
